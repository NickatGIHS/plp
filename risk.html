Risk Taker Goal

For my Risk Taker goal, I have decided to make a website for a comic that I started when I was younger, Silly Stik.
The site will be coded from scratch by me, in PHP (a Hypertext Preprocessor which is basically HTML with more features).

PLAN

I have been toying with the idea of creating a website for Silly Stik for a while now. I think that doing this for PLP has helped me feel like I can do it.

I, like with my Change Maker goal, really just went into this without much of a plan. Once again, I just went with things as they came.

Some notes about the site format

At the start, I decided that ideally for the backend of the website, I would only have to add a file or two (the comic and data for the comic), and the website would add it automatically, as shown in this quick drawing.

I feel like I should also mention that the site format will probably be loosely based off xkcd's website.

This is an email interaction with my brother, the co-creator of Silly Stik

Hi Sam,
I have some questions for you about the Silly Stik Website (http://sillystik.com/).
I would appreciate you please having a look at the site and taking the time to answer the following questions.
QUESTIONS:
As the co-creator of Silly Stik...
  • What are your impressions of seeing Silly Stik as a webcomic?
  • What current features do you like about the site?
  • What features don't you like about the site?
  • What improvements would you suggest for the site?
  • Do you think that the site should be kept running for more than a year (the current paid term is a year)? Why/why not?
  • Is there anything else you'd like to say about the site?
Thanks,
From Nick L
(In 10.01 but also Year 12 Maths Methods)

What are my impressions of seeing Silly Stik as a webcomic?
I think that it is great that we have the opportunity to share these comics that we created as small children to the world, and have a place where we can see them all at once.
What current features of the website do I like?
I like the gallery and the easy navigation as they help with getting around and make the experience much better. Also, the mouseover text is often amusing and a nice touch.
What features don't I like about the website?
I really love the layout and structure of the website, and I don't dislike anything about it. Apart from my MS paint self portrait on the about page, of course. I can never seem to get it just right...
What improvements do I suggest for the website?
If there is one improvement I would suggest, it would be moving some of the text on the homepage to the about page. Apart from that, I really like it and the website is great!
Do I think that the website should be kept running after the year's paid-term?
I think it should be kept running, both to keep these in an accessible archive and in case we make any more and want to upload them.
Is there anything I would like to say about the site?
Not really. I have pretty much said everything I wanted to say in the above questions. I really do like the website, though!

One final note; as Sam has mentioned, he would like to keep the site running for more than the year we have paid for.

A solution to this is migrating the site from HostPapa to GitHub Pages, and changing the PHP code to Javascript. This is a project for the future, but if you wish to access the Silly Stik Site in 2021 or later and sillystik.com does not work, try silly-stik.github.io.
